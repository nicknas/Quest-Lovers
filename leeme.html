Vistas

Nuestras vistas se dividen según el tipo de usuario, por lo que hay unas vistas comunes y unas específicas:

Comunes

Log in/Sign in

Log out

Mi cuenta

Vista con campos de texto que contienen los datos del usuario y que pueden ser modificados. Dos botones de guardar y cancelar, y otros dos para agregar la foto de perfil (seleccionar archivo y enviar).

Específicas

USER

Mis quest

Lista de todos los quests disponibles

Mis mensajes

Lista las conversaciones abiertas con otros usuarios

Matches

Lista de todos los matches del usuario. Al acceder a uno de ellos cambia a la vista del usuario elegido. Esta vista contiene mas informacion y dos botones; "enviar mensaje" que accede a un chat y "reportar usuario" que despliega otro campo de texto y otro boton para realizar un reporte.


ADMIN

Quests

Permite al administrador ver todas las quest disponibles

Editores

Muestra una tabla con los editores registrados, un boton para eliminar uno o varios a la vez y otro boton que despliega un formulario para registrar un nuevo editor.

Reportes

Lista los reportes pendientes y resueltos con desplegables con el nombre del usuario y dentro la descripcion y el estado del reporte.


EDITOR

Mis historias

Permite al editor ver todas las quest que ha creado, borrarlas y editarlas

Editar historia

Página usada para editar cada quest que haya creado ese editor

Subir historia

Vista con dos campos de texto para introducir el nombre y la descripcion de la quest, un boton para seleccionar el archivo ".json" que lleva la quest y un ultimo boton de guardar.

Recursos externos

LocalData

Se guardan fotos y quest(json). La dirección de LocalData es /tmp/iw. Existe en el proyecto una carpeta llamada iw localizada justo en la carpeta principal iw-base en la que se encuentran fotos y quest para testear las subidas a LocalData

Bases de datos

Se usa el perfil default para cargar la base de datos en memoria con el archivo import.sql y el perfil dockerdb para usar la base de datos que tenemos alojada en la dirección jdbc:hsqldb:hsql://container.fdi.ucm.es:20129/iwdb

Contraseñas

Cada usuario tiene como contraseña su nombre de usuario
